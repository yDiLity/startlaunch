# Документация UI компонентов AutoLaunch

## Обзор

Пользовательский интерфейс AutoLaunch реализован с использованием React и Tailwind CSS. Все компоненты разработаны в соответствии с требованиями спецификации и обеспечивают полный функционал для работы с приложением.

## Компоненты

### 1. App.tsx - Главное окно приложения

**Назначение:** Основной компонент приложения, содержащий поле ввода URL и координирующий работу всех остальных компонентов.

**Функциональность:**
- Поле ввода GitHub URL (Требование 1.1, 1.3)
- Кнопка анализа репозитория
- Отображение информации о проекте
- Управление статусом проекта (запуск, остановка, перезапуск)
- Интеграция с менеджером проектов, настройками и окном процесса

**Ключевые состояния:**
- `url` - введенный URL репозитория
- `projectInfo` - информация о проанализированном проекте
- `projectStatus` - текущий статус запущенного проекта
- `showProcessWindow` - отображение окна процесса
- `showProjectManager` - отображение менеджера проектов
- `showSettings` - отображение окна настроек

### 2. ProcessWindow.tsx - Окно процесса с прогресс-баром и логами

**Назначение:** Отображение процесса установки и запуска проекта в реальном времени.

**Требования:** 5.1, 5.2, 5.3, 5.5

**Функциональность:**

#### 5.1 - Прогресс-бар с текущим этапом
- Визуальный индикатор прогресса (0-100%)
- Отображение текущего этапа выполнения:
  - `cloning` - Клонирование репозитория
  - `analyzing` - Анализ проекта
  - `installing` - Установка зависимостей
  - `configuring` - Настройка окружения
  - `starting` - Запуск приложения
  - `running` - Приложение запущено
  - `stopping` - Остановка приложения
  - `stopped` - Приложение остановлено
  - `failed` - Ошибка выполнения

#### 5.2 - Логи в реальном времени
- Отображение логов установки зависимостей
- Автоматическая прокрутка к последним логам
- Возможность ручной прокрутки с отключением автопрокрутки
- Цветовая индикация уровня логов:
  - `info` - информационные сообщения (серый)
  - `warning` - предупреждения (желтый)
  - `error` - ошибки (красный)
  - `success` - успешные операции (зеленый)

#### 5.3 - Понятные сообщения об ошибках
- Отображение ошибки с описанием проблемы
- Предложения по решению проблемы
- Визуальное выделение ошибок

#### 5.5 - Статус "Готово"
- Зеленая галочка при успешном завершении
- Сообщение "Готово! Приложение успешно запущено"

**Интерфейс:**
```typescript
interface ProcessWindowProps {
  projectName: string;           // Имя проекта
  currentStage: ProcessStage;    // Текущий этап
  progress: number;              // Прогресс 0-100
  logs: LogEntry[];              // Массив логов
  error?: string | null;         // Сообщение об ошибке
  errorSuggestion?: string | null; // Предложение по решению
  onClose?: () => void;          // Закрытие окна
  onStop?: () => void;           // Остановка процесса
  isClosable?: boolean;          // Можно ли закрыть окно
}
```

**Использование:**
```tsx
<ProcessWindow
  projectName="owner/repo"
  currentStage="installing"
  progress={45}
  logs={[
    { timestamp: "2024-01-01T12:00:00Z", level: "info", message: "Установка зависимостей..." },
    { timestamp: "2024-01-01T12:00:05Z", level: "success", message: "Зависимости установлены" }
  ]}
  error={null}
  errorSuggestion={null}
  onClose={() => setShowProcessWindow(false)}
  onStop={handleStop}
  isClosable={true}
/>
```

### 3. ProjectManager.tsx - Менеджер проектов

**Назначение:** Управление историей запущенных проектов с возможностью поиска, фильтрации и организации.

**Требования:** 8.1, 8.2, 8.3, 8.4, 8.5

**Функциональность:**
- Отображение списка всех проектов с датами последнего запуска (8.2)
- Поиск проектов по имени, владельцу или стеку (8.3)
- Фильтрация по тегам (8.4)
- Добавление и удаление тегов для организации проектов (8.4)
- Быстрый перезапуск проекта из истории (8.5)
- Удаление проектов из истории

**Интерфейс:**
```typescript
interface ProjectManagerProps {
  onClose: () => void;                    // Закрытие менеджера
  onLaunchProject: (projectId: string) => void; // Запуск проекта
}
```

### 4. Settings.tsx - Окно настроек

**Назначение:** Настройка поведения приложения и пользовательских предпочтений.

**Требования:** 9.1, 9.2, 9.3, 9.4, 9.5

**Функциональность:**

#### 9.1 - Режим изоляции по умолчанию
- Выбор между режимом песочницы (Docker) и прямым режимом
- Применяется ко всем новым проектам

#### 9.2 - Путь сохранения снимков
- Настройка директории для хранения снимков проектов
- Кнопка "Обзор" для выбора директории
- Применяется немедленно ко всем последующим операциям

#### 9.3 - Тема оформления
- Светлая тема
- Темная тема
- Системная тема (следует за настройками ОС)
- Применяется ко всему интерфейсу

#### 9.4 - Автоматическая очистка
- Включение/отключение автоочистки временных файлов
- Настройка максимального возраста снимков (в днях)
- Автоматическое удаление после остановки проектов

#### 9.5 - Сохранение настроек
- Сохранение в конфигурационный файл
- Восстановление при следующем запуске
- Кнопка "Сбросить к умолчаниям"

**Интерфейс:**
```typescript
interface SettingsProps {
  onClose?: () => void; // Закрытие окна настроек
}

interface AppSettings {
  default_isolation_mode: 'Sandbox' | 'Direct';
  snapshots_path: string;
  theme: 'Light' | 'Dark' | 'System';
  auto_cleanup: boolean;
  max_snapshot_age_days: number;
  enable_logging: boolean;
}
```

### 5. SecurityWarnings.tsx - Предупреждения безопасности

**Назначение:** Отображение предупреждений безопасности и управление доверенными репозиториями.

**Требования:** 4.3, 4.4

**Функциональность:**
- Отображение предупреждений безопасности с уровнями (Low, Medium, High, Critical)
- Предложения по решению проблем безопасности
- Добавление репозитория в список доверенных
- Визуальная индикация доверенных репозиториев

**Интерфейс:**
```typescript
interface SecurityWarningsProps {
  warnings: SecurityWarning[];      // Массив предупреждений
  onTrustRepository?: () => void;   // Добавить в доверенные
  isTrusted?: boolean;              // Статус доверия
}

interface SecurityWarning {
  level: "Low" | "Medium" | "High" | "Critical";
  message: string;
  suggestion: string | null;
}
```

## Интеграция компонентов

### Главное окно (App.tsx)

Главное окно координирует работу всех компонентов:

1. **Поле ввода URL** → Анализ → **ProcessWindow** (показывает прогресс)
2. После анализа → Отображение **SecurityWarnings** (если есть предупреждения)
3. Кнопка "История" → Открытие **ProjectManager**
4. Кнопка "Настройки" → Открытие **Settings**
5. Кнопка "Запустить" → **ProcessWindow** (показывает установку и запуск)

### Потоки данных

```
User Input (URL)
    ↓
App.tsx (handleAnalyze)
    ↓
ProcessWindow (analyzing stage)
    ↓
Backend Analysis
    ↓
SecurityWarnings (if warnings exist)
    ↓
User Action (Start)
    ↓
ProcessWindow (installing → starting → running)
    ↓
Project Running
```

## Стилизация

### Tailwind CSS классы

Все компоненты используют Tailwind CSS для стилизации:
- `bg-gray-800` - фон компонентов
- `text-white` - основной текст
- `text-gray-400` - вторичный текст
- `border-gray-700` - границы
- `hover:bg-gray-700` - эффекты наведения
- `transition-colors` - плавные переходы

### Цветовая схема

- **Успех:** `text-green-400`, `bg-green-600`
- **Ошибка:** `text-red-400`, `bg-red-600`
- **Предупреждение:** `text-yellow-400`, `bg-yellow-600`
- **Информация:** `text-blue-400`, `bg-blue-600`

### Темы (Settings.css)

Поддержка светлой и темной темы через CSS переменные:
```css
:root {
  --bg-primary: #1f2937;
  --text-primary: #ffffff;
  /* ... */
}

[data-theme="light"] {
  --bg-primary: #f3f4f6;
  --text-primary: #111827;
  /* ... */
}
```

## Адаптивность

Все компоненты адаптивны и корректно отображаются на разных размерах экрана:
- Мобильные устройства: одноколоночная компоновка
- Планшеты: двухколоночная компоновка
- Десктоп: полная компоновка с максимальной шириной

## Доступность

- Все интерактивные элементы имеют `aria-label`
- Кнопки имеют `title` для подсказок
- Цветовые индикаторы дополнены текстовыми метками
- Поддержка навигации с клавиатуры

## Тестирование

Для тестирования UI компонентов рекомендуется:
1. Проверить все состояния ProcessWindow (все этапы)
2. Проверить отображение ошибок с предложениями
3. Проверить работу поиска и фильтрации в ProjectManager
4. Проверить применение настроек в Settings
5. Проверить отображение предупреждений безопасности всех уровней
