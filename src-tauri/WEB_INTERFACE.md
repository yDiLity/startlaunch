# Веб-интерфейс и детекция портов

## Обзор

Модуль веб-интерфейса обеспечивает автоматическое определение портов запущенных веб-приложений и автоматическое открытие браузера для взаимодействия с ними.

## Требования

Реализует **Требование 10** из спецификации:
- 10.1: Автоматическое определение порта из логов или конфигурации
- 10.2: Автоматическое открытие браузера на localhost:port
- 10.3: Корректное обнаружение нестандартных портов
- 10.4: Показ сообщения с инструкциями при проблемах запуска
- 10.5: Независимость процессов от UI (закрытие браузера не останавливает приложение)

## Архитектура

### Компоненты

#### 1. ProcessController - Детекция портов

**Методы:**

```rust
pub async fn detect_application_port(&self, handle: &ProcessHandle) -> Result<Option<u16>>
```
Автоматически определяет порт приложения из логов процесса.

```rust
fn extract_port_from_log(&self, message: &str) -> Option<u16>
```
Извлекает номер порта из строки лога, поддерживая множество форматов:
- `listening on port 3000`
- `http://localhost:8080`
- `127.0.0.1:5000`
- `started on :4200`
- И другие распространенные паттерны

```rust
pub async fn open_browser_for_port(&self, port: u16) -> Result<()>
```
Открывает браузер по умолчанию на указанном порту. Кроссплатформенная реализация:
- Windows: использует `cmd /C start`
- macOS: использует `open`
- Linux: использует `xdg-open`

```rust
pub async fn check_port_availability(&self, port: u16, timeout_secs: u64) -> Result<bool>
```
Проверяет доступность порта, пытаясь установить TCP соединение.

#### 2. Commands - API для фронтенда

**Команды Tauri:**

```rust
#[tauri::command]
pub async fn detect_and_open_browser(project_id: String) -> Result<serde_json::Value>
```
Комплексная команда, которая:
1. Определяет порт приложения
2. Проверяет его доступность
3. Открывает браузер при успехе
4. Возвращает информативное сообщение при ошибке

```rust
#[tauri::command]
pub async fn check_port_status(port: u16) -> Result<bool>
```
Проверяет доступность конкретного порта.

```rust
#[tauri::command]
pub async fn open_browser_url(url: String) -> Result<()>
```
Открывает браузер на произвольном URL.

## Поток работы

### Автоматический запуск с открытием браузера

```
1. Пользователь запускает проект
   ↓
2. start_project_impl создает процесс
   ↓
3. Система определяет порт из команды/конфигурации
   ↓
4. Ожидание 2 секунды для запуска приложения
   ↓
5. Проверка доступности порта (таймаут 10 сек)
   ↓
6. При успехе: автоматическое открытие браузера
   ↓
7. Возврат статуса пользователю
```

### Ручное открытие браузера

```
1. Пользователь нажимает кнопку "Открыть в браузере"
   ↓
2. Вызов detect_and_open_browser(project_id)
   ↓
3. Детекция порта из логов процесса
   ↓
4. Проверка доступности (таймаут 30 сек)
   ↓
5. Открытие браузера или показ сообщения об ошибке
```

## Детекция портов

### Стратегия определения портов

1. **Из команды запуска** (приоритет 1)
   - Парсинг флагов: `--port`, `-p`, `PORT=`
   - Примеры: `npm start --port 3000`, `python app.py -p 8080`

2. **Из логов процесса** (приоритет 2)
   - Мониторинг вывода приложения
   - Поддержка множества форматов сообщений
   - Регулярные выражения для извлечения портов

3. **Порты по умолчанию** (приоритет 3)
   - Node.js: 3000
   - Python: 5000
   - Общий: 8000

### Поддерживаемые паттерны логов

```
listening on port 3000
server running on port 8080
localhost:4200
127.0.0.1:5000
0.0.0.0:8888
http://example.com:3000
https://example.com:3443
started on :9000
running at http://localhost:7777
available on http://0.0.0.0:4567
```

## Обработка ошибок

### Сценарии ошибок

**1. Порт не определен**
```json
{
  "success": false,
  "message": "Не удалось определить порт приложения. Проверьте логи для получения дополнительной информации."
}
```

**2. Порт недоступен**
```json
{
  "success": false,
  "port": 3000,
  "message": "Приложение не запустилось на ожидаемом порту 3000. Проверьте логи приложения для диагностики проблемы. Возможно, порт занят другим приложением или требуется дополнительная настройка."
}
```

**3. Успешное открытие**
```json
{
  "success": true,
  "port": 3000,
  "url": "http://localhost:3000",
  "message": "Браузер открыт на http://localhost:3000"
}
```

## Независимость процессов

### Требование 10.5: Процессы не зависят от UI

Реализация обеспечивает полную независимость:

1. **Процессы приложений** запускаются как отдельные системные процессы
2. **Браузер** открывается как независимый процесс
3. **Закрытие браузера** не влияет на работу приложения
4. **Закрытие AutoLaunch** предлагает остановить процессы (опционально)

### Жизненный цикл процесса

```
Запуск приложения → Процесс работает в фоне
                           ↓
                    Открытие браузера (независимый процесс)
                           ↓
                    Пользователь закрывает браузер
                           ↓
                    Процесс приложения продолжает работу
                           ↓
                    Явная остановка через UI или stop_project
```

## Примеры использования

### Frontend (TypeScript/JavaScript)

```typescript
// Автоматическое определение и открытие браузера
const result = await invoke('detect_and_open_browser', { 
  projectId: 'project-uuid' 
});

if (result.success) {
  console.log(`Браузер открыт: ${result.url}`);
} else {
  console.error(`Ошибка: ${result.message}`);
}

// Проверка доступности порта
const isAvailable = await invoke('check_port_status', { 
  port: 3000 
});

// Открытие произвольного URL
await invoke('open_browser_url', { 
  url: 'http://localhost:8080/admin' 
});
```

### Backend (Rust)

```rust
// Детекция порта из handle
let port = process_controller
    .detect_application_port(&handle)
    .await?;

// Проверка доступности
if let Some(port) = port {
    let is_available = process_controller
        .check_port_availability(port, 10)
        .await?;
    
    if is_available {
        process_controller
            .open_browser_for_port(port)
            .await?;
    }
}
```

## Кроссплатформенность

### Windows
- Открытие браузера: `cmd /C start <url>`
- Поддержка всех основных браузеров через системные настройки

### macOS
- Открытие браузера: `open <url>`
- Использует браузер по умолчанию из системных настроек

### Linux
- Открытие браузера: `xdg-open <url>`
- Поддержка различных окружений рабочего стола (GNOME, KDE, XFCE и др.)

## Ограничения и известные проблемы

1. **Детекция портов из логов**
   - Зависит от формата вывода приложения
   - Может не работать для приложений с нестандартным форматом логов
   - Решение: добавление новых паттернов по мере необходимости

2. **Таймауты**
   - Автоматическое открытие: 10 секунд ожидания
   - Ручное открытие: 30 секунд ожидания
   - Медленные приложения могут не успеть запуститься

3. **Множественные порты**
   - Если приложение слушает несколько портов, открывается только первый
   - Решение: UI для выбора порта
