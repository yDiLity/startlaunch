# Система логирования AutoLaunch

## Обзор

AutoLaunch использует библиотеку `tracing` для структурированного логирования. Система логирования настроена для предоставления детальной информации о работе приложения в режиме разработки и оптимизированного вывода в продакшене.

## Конфигурация

### Уровни логирования

Система поддерживает следующие уровни логирования (от наименее к наиболее важным):

- **TRACE**: Очень детальная информация для отладки
- **DEBUG**: Отладочная информация
- **INFO**: Информационные сообщения о нормальной работе (по умолчанию)
- **WARN**: Предупреждения о потенциальных проблемах
- **ERROR**: Ошибки, требующие внимания

### Настройки вывода

Текущая конфигурация в `main.rs`:

```rust
tracing_subscriber::fmt()
    .with_max_level(tracing::Level::INFO)  // Максимальный уровень
    .with_target(false)                     // Не показывать target модуля
    .with_thread_ids(true)                  // Показывать ID потоков
    .with_file(true)                        // Показывать имя файла
    .with_line_number(true)                 // Показывать номер строки
    .init();
```

## Использование в коде

### Основные макросы

```rust
use tracing::{info, debug, warn, error, trace};

// Информационное сообщение
tracing::info!("Запуск приложения");

// Сообщение с параметрами
tracing::info!("Анализ проекта: {}", project_name);

// Отладочная информация
tracing::debug!("Обработка файла: {:?}", file_path);

// Предупреждение
tracing::warn!("Не удалось найти конфигурационный файл");

// Ошибка
tracing::error!("Критическая ошибка: {}", error);
```

### Структурированное логирование

```rust
// С несколькими полями
tracing::info!(
    project_id = %project.id,
    repo_name = %project.repo_name,
    "Проект сохранен в базу данных"
);

// С контекстом
tracing::info!(
    owner = %owner,
    repo = %repo_name,
    url = %normalized_url,
    "URL распознан"
);
```

## Логирование по модулям

### main.rs
- Запуск приложения
- Инициализация компонентов
- Критические ошибки

### database.rs
- Подключение к базе данных
- Выполнение миграций
- Операции с данными (сохранение, чтение, удаление)

### commands.rs
- Начало и завершение команд
- Парсинг URL
- Клонирование репозиториев
- Анализ проектов

### project_analyzer.rs
- Сканирование директорий
- Детекция стеков технологий
- Парсинг конфигурационных файлов

### environment_manager.rs
- Создание окружений
- Сборка Docker образов
- Установка зависимостей

### process_controller.rs
- Запуск процессов
- Мониторинг статуса
- Остановка процессов

## Настройка уровня логирования

### Через переменную окружения

```bash
# Linux/macOS
export RUST_LOG=debug
npm run tauri dev

# Windows PowerShell
$env:RUST_LOG="debug"
npm run tauri dev

# Windows CMD
set RUST_LOG=debug
npm run tauri dev
```

### Уровни по модулям

```bash
# Только INFO для всего, но DEBUG для database
export RUST_LOG=info,autolaunch::database=debug

# TRACE для конкретного модуля
export RUST_LOG=autolaunch::project_analyzer=trace
```

## Логирование в продакшене

В продакшене рекомендуется:

1. Установить уровень `INFO` или `WARN`
2. Настроить вывод в файл (будет реализовано позже)
3. Использовать ротацию логов
4. Отправлять критические ошибки в систему мониторинга
