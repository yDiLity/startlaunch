# Active Context

## Текущий фокус

Завершена синхронизация основного frontend flow с Rust backend: UI использует реальный `project_id`, реальные логи процессов и backend-driven restart/status flow. Текущий фокус смещён на поддержку канонической документации, уточнение ограничений окружения и финальную верификацию Rust/Tauri-проверок.

## Статус проекта

- ✅ Основной пользовательский сценарий анализа и запуска синхронизирован между UI и backend
- ✅ Из основного UI flow убраны следы симуляции запуска
- ⚠️ Документация приведена к каноническому виду, но Rust/Tauri verification всё ещё ограничен окружением
- ❌ Tauri сборка в текущем окружении требует Visual Studio

## Следующие шаги

1. Прогнать `bun`, Biome и Rust-проверки после синхронизации UI и документации
2. Зафиксировать ограничения Windows-сборки Tauri в пользовательской и внутренней документации
3. При необходимости декомпозировать `src/App.tsx` на более мелкие orchestration hooks/components
4. Рассмотреть переход с polling на event-driven доставку логов и статусов
5. Поддерживать `docs/README.md` как единственный источник high-level архитектуры

## Активные решения

- Канонический контракт анализа будет возвращать и `project_id`, и данные проекта
- Перезапуск должен выполняться backend-командой, а не UI-таймером
- При отсутствии автоопределённой команды запуска пользователь должен иметь ручной override
- Пользовательская документация и Tauri-конфиг должны использовать `bun` как единственный JS package manager
- D14 нельзя считать завершённым, пока Rust/Tauri проверки не подтверждены в рабочем окружении
