# Active Context

## Текущий фокус

В работе доведение проекта до максимально возможной готовности в текущем окружении: синхронизация frontend и Rust backend, удаление mock-сценариев из основного флоу анализа/запуска, возврат реального `project_id`, подключение реальных логов процессов, ручного ввода команды запуска, перевод команд на `bun` и приведение документации к фактическому состоянию проекта.

## Статус проекта

- ⚠️ Основной пользовательский сценарий частично рассинхронизирован между UI и backend
- ⚠️ В интерфейсе есть следы симуляции прогресса и запуска
- ⚠️ Документация и Memory Bank требуют синхронизации с кодом
- ❌ Tauri сборка в текущем окружении требует Visual Studio

## Следующие шаги

1. Завершить интеграцию реального backend flow в `src/App.tsx` и `src-tauri/src/commands.rs`
2. Подключить реальные логи процессов, корректный restart/status flow и ручной override команды запуска
3. Перевести пользовательские и системные команды проекта с `npm` на `bun`
4. Синхронизировать `docs/README.md`, `README.md` и `memory_bank/*`
5. Прогнать `bun`/Biome/Rust проверки и отдельно зафиксировать ограничения окружения

## Активные решения

- Канонический контракт анализа будет возвращать и `project_id`, и данные проекта
- Перезапуск должен выполняться backend-командой, а не UI-таймером
- При отсутствии автоопределённой команды запуска пользователь должен иметь ручной override
- Пользовательская документация и Tauri-конфиг должны использовать `bun` как единственный JS package manager
- Процент готовности проекта нельзя считать достоверным до синхронизации deliverables с кодом
