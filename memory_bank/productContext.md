# Product Context

## Проблема

Разработчики, студенты и исследователи теряют время при запуске проектов с GitHub:
- Плохая или устаревшая документация
- Конфликты зависимостей с основной системой
- Сложные шаги настройки окружения
- Страх командной строки у новичков

## Решение

AutoLaunch автоматизирует весь pipeline: клонирование → анализ → настройка окружения → установка зависимостей → запуск → предоставление доступа.

## Ключевые пользовательские сценарии

1. Новичок вставляет ссылку на React-проект → видит его в браузере без знания Webpack/Node
2. Инженер тестирует Rust CLI-утилиту без установки компилятора в основную систему
3. Преподаватель даёт студентам одну ссылку — все получают одинаковое окружение
4. Security-исследователь безопасно запускает подозрительный скрипт в изолированной среде

## Конкурентный контекст

Текущие решения (Docker вручную, ручная настройка) требуют технических знаний. AutoLaunch — единый простой интерфейс поверх них.

## Бизнес-цель

Стать стандартным инструментом в workflow разработчиков при тестировании open-source проектов.
