#### Снимки проектов

### Обработка ошибок на уровне приложения

Все команды используют единую систему обработки ошибок:

```rust
pub struct ErrorContext {
    pub error: AutoLaunchError,
    pub suggestion: Option<String>,
    pub recovery_actions: Vec<RecoveryAction>,
    pub user_friendly_message: String,
}
```

Каждая команда:
1. Выполняет операцию в `_impl` функции
2. Возвращает `Result<T, AutoLaunchError>`
3. Конвертирует ошибки в `ErrorContext` для фронтенда
4. Предоставляет понятные сообщения и предложения решений

## 4. Основные сценарии использования

### Сценарий 1: Запуск нового проекта
1. Пользователь вводит URL репозитория
2. Frontend вызывает `analyze_repository`
3. Backend клонирует репозиторий
4. ProjectAnalyzer определяет стек и команды
5. SecurityScanner проверяет безопасность
6. Данные сохраняются в БД
7. Frontend отображает информацию
8. Пользователь нажимает "Запустить"
9. Backend создает окружение и запускает процесс
10. Frontend показывает статус и логи

### Сценарий 2: Работа со снимками
1. Проект успешно запущен
2. Пользователь сохраняет снимок
3. SnapshotManager сериализует окружение
4. Данные сохраняются в БД
5. При следующем запуске загружается снимок
6. Проект запускается быстрее (без установки зависимостей)

### Сценарий 3: Управление безопасностью
1. При анализе проекта сканируются команды
2. Обнаруженные угрозы показываются пользователю
3. Пользователь может добавить репозиторий в доверенные
4. Статус доверия сохраняется в БД
5. При следующем запуске предупреждения не показываются
